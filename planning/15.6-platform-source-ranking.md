# macOS 15.6 Platform Source Ranking

Use this note when choosing the next `15.6` acquisition step for platform-facing research.

## Ranking

| Rank | Source Target | Unlocks | Boundary |
| --- | --- | --- | --- |
| 1 | Verified `15.6` mounted system volume or user-provided installer with accessible extension `Info.plist` files | Personality extraction, PCIe metadata, provider-class candidates, controller-family backlog | Commit reduced metadata only. |
| 2 | Official Apple `15.6` installer acquired into `downloads/` and reduced locally | Same as rank 1, plus stronger reproducibility if acquisition command is documented | Do not commit installer payloads. |
| 3 | Verified `15.6` host runtime with read-only local commands | Runtime facts, topology samples, local provenance | Must label as host-specific runtime evidence. |
| 4 | Current host runtime | Tool smoke checks only | Rejected as `15.6` evidence because host reports `26.5.1` build `25F80`. |

## Best First Reduced Output

The highest-value first output is a small kext or DriverKit personality dataset from a verified `15.6` source that includes:

- bundle identifier;
- bundle type;
- personality name;
- `IOClass`;
- `IOProviderClass`;
- PCI match keys when present;
- source path;
- acquisition date;
- redaction policy;
- validation command.

## Why This Source Comes First

A verified personality source can feed:

- `knowledge/platform/matching-path.md`;
- `knowledge/platform/candidate-joins.md`;
- `knowledge/platform/pcie-metadata-coverage.md`;
- `knowledge/platform/controller-family-backlog.md`;
- future release-diff readiness checks.

It does not require raw boot artifacts, raw IORegistry dumps, binary payloads, or runtime attachment claims.

## Stop Conditions

Stop before deriving metadata when:

- the source cannot be proven to be `15.6`;
- provenance is missing;
- redaction cannot be reviewed;
- only raw payloads are available;
- the step would require boot modification, patching, or unsupported hardware claims.
