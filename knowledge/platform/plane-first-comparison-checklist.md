# Plane First Comparison Checklist

## Summary

Registry comparison should record plane facts before comparing families. Without plane context, class names and paths can be misread as equivalent relationships.

## Context

- Related registry note: `knowledge/iokit/IORegistry.md`
- Related topology note: `knowledge/platform/topology-vocabulary.md`
- Related schema: `datasets/schemas/ioreg-inventory.schema.json`
- Queue cycle: `planning/PLATFORM_SESSION_QUEUE.md` cycle 245

## Checklist

- [ ] Record plane name.
- [ ] Record source command or artifact.
- [ ] Record class name and depth.
- [ ] Record whether object names passed redaction review.
- [ ] Avoid cross-plane path comparisons unless the relationship is defined.
