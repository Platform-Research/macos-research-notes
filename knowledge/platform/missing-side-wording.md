# Missing Side Wording

## Summary

When one side of a release comparison is missing, the note may describe the missing input but should not describe the release as changed. Missing-side wording preserves the acquisition gap without inventing a diff.

- Related fixture: `datasets/fixtures/release-diff/missing-artifacts-15.6-to-16.0.fixture.json`

- Related readiness guide: `docs/guidelines/release-diff-input-readiness.md`
- Queue cycle: `planning/PLATFORM_SESSION_QUEUE.md` cycle 253

## Preferred Wording

- "missing comparable input"
- "unresolved"
- "not acquired"
- "no change can be inferred"
